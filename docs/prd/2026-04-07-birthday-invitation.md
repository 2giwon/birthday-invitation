## 1. 개요

### 1.1 핵심 문제
2026년 4월 11일 이지호 학생의 생일 파티에 친구들을 초대하기 위한 디지털 초대장이 필요하다. 카카오톡으로 URL을 공유하면 모바일에서 파티 정보(일시, 장소, 집합, 준비물)를 한 눈에 확인할 수 있어야 한다.

### 1.2 성공 기준
- [ ] 카카오톡으로 URL 공유 시 OG 미리보기(제목, 설명, 썸네일)가 정상 노출된다
- [ ] 카카오톡 인앱 브라우저(iOS/Android)에서 모든 정보가 정상 표시된다
- [ ] 파티 정보를 세로 스크롤 한 방향으로 모두 확인할 수 있다

### 1.3 대상 사용자

| 페르소나 | 설명 | 주요 니즈 |
|---------|------|----------|
| 친구 (초등학생 5학년) | 카카오톡 단체방에서 링크 수신, 스마트폰 접속 | 파티 시간/장소/준비물 빠르게 확인 |
| 친구 부모님 | 카카오톡에서 링크 수신 | 집합/파티 장소 지도 확인, 보호자 연락 |
| 보호자 (이지호 엄마) | 카카오톡으로 URL 공유 | 파티 정보 정확 노출 확인, 예쁜 미리보기 |

## 3. 기능 요구사항

### F-1: 페이지 레이아웃 및 섹션 구조 (Must)

모바일 청첩장 스타일의 세로 스크롤 단일 페이지를 구현한다.

**섹션 순서:**

```
1. 히어로 섹션     — 이미지 + "생일에 초대합니다!" 문구
2. 인사말 섹션     — "지호의 생일에 함께해 주세요!"
3. 일시 섹션       — 2026년 4월 11일 (금) 오후 2시
4. 집합 안내 섹션  — 오후 1시, 지호네 집 앞 + 네이버맵 링크
5. 파티 장소 섹션  — 원마운트 1층 + 네이버맵 링크
6. 준비물 섹션     — 미끄럼 방지용 양말
7. 연락처/푸터     — 엄마 연락처 + 전화 걸기 버튼
```

**수용 기준:**
- [ ] 7개 섹션이 위 순서대로 세로 스크롤로 배치된다
- [ ] 최대 너비 480px, 화면 중앙 정렬
- [ ] 섹션별 배경색 또는 카드 형태로 시각적 구분이 있다
- [ ] 320px 뷰포트(iPhone SE)에서 콘텐츠 잘림 없이 표시된다
- [ ] 데스크톱에서 접속해도 레이아웃이 깨지지 않는다 (중앙 정렬, 최대 너비 제한)
- [ ] [비정상] 뷰포트가 480px 이상일 때 콘텐츠가 480px 내에 유지된다

### F-2: 파티 정보 표시 (Must)

각 섹션에 확정된 파티 정보를 정확하게 표시한다.

**수용 기준:**
- [ ] 일시 섹션에 "2026년 4월 11일 (금) 오후 2시"가 표시된다
- [ ] 집합 안내 섹션에 "오후 1시"와 "서울시 강서구 양천로 62길 41 코오롱오투빌2차 아파트 앞"이 표시된다
- [ ] 파티 장소 섹션에 "원마운트 1층"과 "경기 고양시 일산서구 한류월드로 300"이 표시된다
- [ ] 준비물 섹션에 "미끄럼 방지용 양말"이 표시된다
- [ ] 인사말 섹션에 "지호의 생일에 함께해 주세요!"가 표시된다
- [ ] 본문 텍스트의 최소 폰트 크기는 16px이다 (보조 텍스트인 주소/캡션은 14px 허용)
- [ ] [비정상] 웹폰트(Noto Sans KR) 로딩 실패 시 시스템 폰트로 폴백되어 정상 표시된다 (font-display: swap)

### F-3: 네이버맵 지도 링크 (Must)

집합 장소와 파티 장소 각각에 네이버맵 웹 링크 버튼을 제공한다.

**네이버맵 URL (검색 쿼리 기반):**
- 집합 장소: `https://map.naver.com/v5/search/서울시 강서구 양천로 62길 41`
- 파티 장소: `https://map.naver.com/v5/search/경기 고양시 일산서구 한류월드로 300 원마운트`

**수용 기준:**
- [ ] 집합 안내 섹션에 "지도 보기" 버튼이 있고, 탭 시 네이버맵 웹 URL로 이동한다
- [ ] 파티 장소 섹션에 "지도 보기" 버튼이 있고, 탭 시 네이버맵 웹 URL로 이동한다
- [ ] 지도 링크는 `https://map.naver.com/v5/search/...` 형태의 웹 URL이다 (앱 딥링크 아님)
- [ ] 버튼의 터치 타겟이 최소 44x44px이다
- [ ] [비정상] 카카오톡 인앱 브라우저에서 링크 탭 시 정상적으로 외부 페이지가 열린다
- [ ] [비정상] 네이버맵 앱 미설치 시에도 웹 브라우저에서 지도가 열린다

### F-4: OG 메타태그 — 카카오톡 미리보기 (Must)

카카오톡에 URL 공유 시 미리보기가 정상 표시되도록 OG 메타태그를 설정한다.

**OG 값:**
```html
<meta property="og:title" content="이지호의 생일 파티에 초대합니다!" />
<meta property="og:description" content="4월 11일(금) 오후 2시, 원마운트에서 만나요!" />
<meta property="og:image" content="{배포URL}/og-image.png" />
<meta property="og:url" content="{배포URL}" />
<meta property="og:type" content="website" />
```

**OG 이미지 규격:**
- 권장 사이즈: 1200x630px (1.91:1 비율)
- 최소 사이즈: 800x400px
- 최대 파일 용량: 300KB
- 로컬 개발 시: 상대 경로 `/og-image.png`로 테스트, 배포 후 절대 URL로 자동 전환

**수용 기준:**
- [ ] HTML `<head>`에 og:title, og:description, og:image, og:url, og:type 메타태그가 존재한다
- [ ] og:title 값이 "이지호의 생일 파티에 초대합니다!"이다
- [ ] og:description 값이 "4월 11일(금) 오후 2시, 원마운트에서 만나요!"이다
- [ ] og:image가 유효한 이미지 URL을 가리킨다
- [ ] OG 이미지 파일이 `public/og-image.png` 경로에 존재한다
- [ ] [비정상] OG 이미지 로딩 실패 시에도 제목/설명은 정상 노출된다
- [ ] [배포 후] 카카오 디버거에서 OG 캐시 초기화 후 미리보기 정상 확인

### F-5: 보호자 연락처 + 전화 걸기 (Should)

페이지 하단에 보호자 연락처와 전화 걸기 버튼을 제공한다.

**수용 기준:**
- [ ] 연락처/푸터 섹션에 "엄마" 레이블과 "010-9921-3109"가 표시된다
- [ ] 연락처 또는 전화 걸기 버튼을 탭하면 전화 앱이 실행된다 (`tel:010-9921-3109`)
- [ ] 전화 걸기 버튼의 터치 타겟이 최소 44x44px이다
- [ ] [비정상] 데스크톱에서는 전화 걸기가 불가하더라도 번호가 텍스트로 확인 가능하다

### F-6: 히어로 이미지 (Should)

초대장 상단에 이미지 영역을 마련하고, 추후 실제 사진으로 교체 가능하게 한다.

**수용 기준:**
- [ ] 히어로 섹션에 이미지가 표시된다
- [ ] 현재는 생일 테마 기본 이미지(케이크/풍선 등)가 표시된다
- [ ] 이미지 파일 경로가 `public/images/hero.jpg`로 분리되어 있어 교체 용이하다
- [ ] 이미지가 모바일 화면 너비에 맞게 표시된다 (가로 잘림 없음, object-fit: cover 또는 contain)
- [ ] [비정상] 이미지 로딩 실패 시 alt 텍스트("이지호의 생일 파티")가 표시되고 레이아웃이 깨지지 않는다

**히어로 이미지 규격:**
- 권장 사이즈: 480x320px (3:2 비율)
- 최대 파일 용량: 500KB
- 포맷: JPG (사진 교체 대비)

### F-7: 스크롤 애니메이션 (Could)

각 섹션이 스크롤에 따라 부드럽게 나타나는 fade-in 애니메이션을 적용한다.

**수용 기준:**
- [ ] 각 섹션이 뷰포트에 진입할 때 fade-in 애니메이션이 적용된다
- [ ] 애니메이션이 과하지 않고 자연스럽다 (duration 0.3~0.6s)
- [ ] [비정상] IntersectionObserver 미지원 브라우저에서는 애니메이션 없이 즉시 표시된다
- [ ] [비정상] prefers-reduced-motion 미디어 쿼리를 존중하여 애니메이션 비활성화 가능

## 4. 비기능 요구사항

### NFR-1: 성능
- [ ] 3G 환경에서도 핵심 텍스트가 3초 이내 표시 (font-display: swap)
- [ ] 총 페이지 용량 3MB 이하 (이미지 포함)
- [ ] Lighthouse Performance 점수 80 이상

### NFR-2: 호환성
- [ ] `<meta name="viewport" content="width=device-width, initial-scale=1.0">` 설정 필수
- [ ] 카카오톡 인앱 브라우저 (iOS/Android) 정상 동작
- [ ] Safari (iOS 15+), Chrome (Android), Samsung Internet 지원
- [ ] 최소 뷰포트: 320px (iPhone SE)
- [ ] 데스크톱 접속 시 레이아웃 깨지지 않음

### NFR-3: 접근성
- [ ] 본문 텍스트(16px 이상) 색상 대비 WCAG AA 기준 충족 (4.5:1 이상)
- [ ] 버튼 텍스트(18px+ 큰 텍스트)는 3:1 이상 (버튼: #333333 on #FFB4A2 ≈ 3.7:1)
- [ ] 모든 이미지에 alt 텍스트 제공
- [ ] 한국어 단일 언어 (`lang="ko"`)
- [ ] favicon 제공 (생일 테마 아이콘, `public/favicon.ico`)

### NFR-4: 보안
- [ ] 보호자 연락처는 tel: 링크로 제공 (난독화 불필요)
- [ ] 민감 정보 최소 노출 (공개 URL)
