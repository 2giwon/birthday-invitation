# 생일 초대장 웹페이지 기획 시나리오

## 프로젝트 개요

초등학교 5학년 이지호 학생의 생일 초대장을 모바일 웹페이지로 제작한다.
모바일 청첩장 스타일의 구성으로, 카카오톡을 통해 URL을 공유하면 누구나 접속하여 초대 정보를 확인할 수 있다.

### 왜 만드는가?
- 2026년 4월 11일 이지호 학생의 생일 파티에 친구들을 초대하기 위한 디지털 초대장
- 카카오톡으로 간편하게 공유할 수 있는 모바일 최적화 초대 페이지
- **마감일**: 2026년 4월 10일까지 배포 완료 (파티 전날)

### 성공 기준
- 카카오톡으로 URL 공유 시 미리보기(OG)가 정상 노출된다
- 모바일(카카오톡 인앱 브라우저)에서 모든 정보가 정상 표시된다
- 파티 정보(일시, 장소, 집합, 준비물)를 한 화면 스크롤로 모두 확인할 수 있다

---

## 대상 사용자 (페르소나)

### 페르소나 1: 초대 수신자 — 친구 (초등학생 5학년)
- 카카오톡 단체방에서 링크를 받음
- 스마트폰으로 접속
- 파티 장소와 시간, 준비물을 빠르게 확인하고 싶음

### 페르소나 2: 초대 수신자 — 친구 부모님
- 카카오톡 개인 메시지 또는 단체방에서 링크를 받음
- 집합 장소(지호네 집 앞)와 파티 장소(원마운트) 위치를 지도로 확인하고 싶음
- 문의 사항이 있으면 보호자(엄마)에게 연락하고 싶음

### 페르소나 3: 초대 발신자 — 보호자 (이지호 엄마)
- 카카오톡으로 초대장 URL을 공유함
- 파티 정보가 정확하게 노출되는지 확인

## User Story

### US-1: 파티 정보 확인 (Must)
> 초대 수신자로서, 카카오톡 링크를 눌러 생일 파티의 일시/장소/집합 정보/준비물을 한 번에 확인하고 싶다. 왜냐하면 여러 곳을 찾아보지 않고 한 페이지에서 모든 정보를 얻고 싶기 때문이다.

**수용 기준:**
- [ ] 페이지에 접속하면 파티 날짜(2026.4.11)와 시간(오후 2시)이 표시된다
- [ ] 파티 장소명(원마운트)과 주소가 표시된다
- [ ] 집합 시간(오후 1시)과 집합 장소(지호네 집 앞 주소)가 표시된다
- [ ] 준비물(미끄럼 방지용 양말)이 표시된다
- [ ] 모든 정보는 세로 스크롤 한 방향으로 확인 가능하다
- [ ] 3G 네트워크에서도 텍스트 정보가 3초 이내 표시된다
- [ ] 웹폰트 로딩 실패 시 시스템 폰트(Apple SD Gothic Neo, sans-serif)로 폴백되어 정상 표시된다

### US-2: 장소 지도 확인 (Must)
> 친구 부모님으로서, 파티 장소와 집합 장소의 위치를 지도로 확인하고 싶다. 왜냐하면 처음 가는 곳일 수 있기 때문이다.

**수용 기준:**
- [ ] 집합 장소(지호네 집 앞) 섹션에 네이버맵 링크 버튼이 있다
- [ ] 파티 장소(원마운트) 섹션에 네이버맵 링크 버튼이 있다
- [ ] 지도 링크는 https:// 웹 URL 형태로 제공되어 앱 미설치 시에도 웹 브라우저에서 접근 가능하다
- [ ] 카카오톡 인앱 브라우저에서 지도 링크 탭 시 정상적으로 외부 페이지가 열린다

### US-3: 보호자 연락 (Should)
> 친구 부모님으로서, 문의 사항이 있을 때 보호자(엄마)에게 전화하고 싶다. 왜냐하면 파티 관련 추가 정보를 확인해야 할 수 있기 때문이다.

**수용 기준:**
- [ ] 페이지 하단에 보호자(엄마) 연락처(010-9921-3109)가 표시된다
- [ ] 연락처를 탭하면 전화 앱이 실행된다 (tel: 링크)
- [ ] 데스크톱에서는 전화 걸기가 불가하더라도 연락처 번호가 텍스트로 확인 가능하다

### US-4: 카카오톡 공유 (Must)
> 보호자로서, 카카오톡에 초대장 URL을 공유했을 때 예쁜 미리보기가 표시되길 원한다. 왜냐하면 링크만 덜렁 보내면 누를 사람이 적기 때문이다.

**수용 기준:**
- [ ] 카카오톡에 URL을 붙여넣으면 OG 미리보기(제목, 설명, 썸네일 이미지)가 표시된다
- [ ] OG 제목: "이지호의 생일 파티에 초대합니다!" (또는 유사 문구)
- [ ] OG 설명: 날짜/시간 요약 정보
- [ ] OG 이미지: 초대장 대표 이미지 — 배포 후 `{배포URL}/og-image.png` 경로
- [ ] OG 이미지 로딩 실패 시에도 제목/설명은 정상 노출된다

### US-5: 히어로 이미지 (Should)
> 초대 수신자로서, 초대장 상단에 이지호 학생의 사진이나 생일 관련 이미지를 보고 싶다. 왜냐하면 초대장이 더 특별하게 느껴지기 때문이다.

**수용 기준:**
- [ ] 히어로 섹션에 이미지 영역이 마련되어 있다 (placeholder 포함)
- [ ] 사진 미제공 시 생일 테마 기본 이미지(케이크/풍선 일러스트)가 표시된다
- [ ] 추후 실제 사진으로 교체 가능하도록 이미지 경로가 분리되어 있다 (`public/images/hero.jpg`)
- [ ] 이미지는 모바일 화면에 맞게 최적화되어 표시된다 (가로 잘림 없음)
- [ ] 이미지 로딩 실패 시 레이아웃이 깨지지 않는다 (alt 텍스트 표시)

## 디자인

### 색상 팔레트 (파스텔톤)
- 메인 배경: 연한 크림/아이보리 (#FFF8F0 또는 유사)
- 포인트 색상: 파스텔 핑크, 파스텔 옐로우, 파스텔 민트 등
- 텍스트: 다크 그레이 (#333333) — 가독성 확보
- 섹션 구분: 부드러운 그라데이션 또는 파스텔 색상 배경 전환

### 레이아웃
- 모바일 청첩장 스타일 (세로 스크롤, 최대 너비 480px, 중앙 정렬)
- 섹션별 구분이 명확한 카드 형태
- 부드러운 스크롤 애니메이션 (fade-in 등, 과하지 않게)

### 폰트
- 한글 웹폰트 (Google Fonts — Noto Sans KR)
- 폴백: "Apple SD Gothic Neo", "Malgun Gothic", sans-serif
- font-display: swap (웹폰트 로딩 전 시스템 폰트로 즉시 표시)
- 최소 본문 폰트 크기: 16px (모바일 가독성)
- 제목은 22~28px

### 이미지
- 히어로 섹션: 추후 이지호 학생 실제 사진으로 교체 가능 (현재는 생일 테마 기본 이미지)
- 이미지 파일 경로: `public/images/hero.jpg` (교체 용이)
- 생일 테마 일러스트/아이콘 (케이크, 풍선 등 — 무료 소스 활용)
- OG 썸네일: `public/og-image.png` (파스텔톤 생일 테마)

### 디자인 레퍼런스
- 모바일 청첩장 서비스(바른, 크라우디, 보헴 등)의 세로 스크롤 레이아웃 참고
- 초등학생 대상이므로 밝고 활기찬 느낌, 생일 파티 분위기

## 비기능 요구사항

### 성능 (CON)
- 3G 환경에서도 핵심 정보(텍스트)가 3초 이내 표시
- 총 페이지 용량 3MB 이하 (이미지 포함)
- Lighthouse Performance 점수 80 이상

### 호환성 (CON)
- 카카오톡 인앱 브라우저 (iOS/Android) 정상 동작
- Safari (iOS 15+), Chrome (Android), Samsung Internet 지원
- 최소 뷰포트: 320px (iPhone SE)
- 데스크톱 접속 시에도 레이아웃 깨지지 않음 (중앙 정렬, 최대 너비 제한)

### 접근성 (GUD)
- 색상 대비 WCAG AA 기준 충족 (4.5:1 이상)
- 이미지에 alt 텍스트 제공
- 한국어 단일 언어

### 보안 (SEC)
- 공개 URL이므로 민감 정보 최소 노출
- 보호자 연락처는 tel: 링크로 제공 (난독화 불필요 — 카카오톡 단체방 공유 한정)

## OG (Open Graph) 메타태그

```html
<meta property="og:title" content="이지호의 생일 파티에 초대합니다!" />
<meta property="og:description" content="4월 11일(금) 오후 2시, 원마운트에서 만나요!" />
<meta property="og:image" content="{배포URL}/og-image.png" />
<meta property="og:type" content="website" />
```

- OG 이미지: `public/og-image.png` (빌드 시 `/og-image.png`로 서빙)
- 배포 후 카카오 OG 캐시 초기화: [카카오 디버거](https://developers.kakao.com/tool/debugger/sharing)에서 URL 입력하여 캐시 갱신

## 기술 스택

- **프레임워크**: Vite + vanilla HTML/CSS/JS (React 불필요 — 단일 정적 페이지)
- **스타일링**: CSS (커스텀, 별도 라이브러리 없음)
- **폰트**: Google Fonts (Noto Sans KR), font-display: swap
- **지도**: 네이버맵 웹 URL 링크 (API 키 불필요, 단순 https 링크)
- **호스팅**: Vercel (무료 Hobby 플랜, 서브도메인 제공, Git 연동 자동 배포, HTTPS 기본 제공)
- **빌드**: Vite (정적 빌드, `dist/` 출력)

## 우선순위

| 우선순위 | Feature |
|---------|---------|
| **Must** | 파티 정보 표시 (일시, 장소, 집합, 준비물) |
| **Must** | 네이버맵 지도 링크 |
| **Must** | OG 메타태그 (카카오톡 미리보기) |
| **Must** | 모바일 반응형 레이아웃 |
| **Should** | 보호자 연락처 + 전화 걸기 |
| **Should** | 히어로 이미지 (추후 사진 교체 가능한 placeholder) |
| **Could** | 스크롤 애니메이션 (fade-in) |
| **Could** | 생일 테마 일러스트/아이콘 |

### 마감 리스크 관리
- 4/9까지 Must/Should 미완료 시 Could는 전부 드롭
- Must만 구현해도 초대장으로서 기능함 (MVP)
