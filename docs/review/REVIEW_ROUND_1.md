## 핵심 피드백 요약

1. **구체적 파티 정보가 없다** — 날짜, 시간, 장소, 주소, 준비물 등 실제 데이터가 플레이스홀더 수준이다.
2. **수용 기준(Acceptance Criteria)이 전무하다** — 3명 만장일치 지적.
3. **RSVP "미정" 상태가 아키텍처를 좌우한다** — 즉시 In/Out 결정 필요.
4. **User Story / 사용 흐름이 없다** — 누가 어떤 경로로 진입하고 어떤 순서로 정보를 확인하는지 미정의.
5. **페이지 섹션 구조가 없다** — 어떤 섹션이 어떤 순서로 배치되는지 미정.
6. **디자인 기준이 주관적이다** — "밝고 파스텔톤"은 검증 불가능.
7. **기술 스택이 미확정이다** — 순수 HTML vs React 미결정.
8. **OG 메타태그/카카오톡 공유 미리보기 미정의** — URL 공유가 핵심인데 관련 요구사항 없음.

## 질문 (사용자에게)

### PM 리뷰어 질문
1. 생일 파티는 정확히 언제인가요? 마감일을 역산해야 합니다.
2. 실제 파티 정보를 알려주실 수 있나요? (날짜, 시간, 장소명, 주소, 집합 장소, 준비물)
3. 초대장을 어떤 방식으로 공유할 예정인가요? (카카오톡, 문자, 학교 단체방 등)
4. RSVP(참석 여부 확인)는 필요한가요?
5. 장소 안내에 지도가 필요한가요?
6. 디자인 레퍼런스가 있나요?
7. 이지호 학생의 사진이나 이미지가 들어가나요?
8. 보호자 연락처가 노출되어야 하나요?
